# V1.1 Models Rewrite Summary

## Problem Identified

The original V1.1 models were trying to merge V1's workflow with v1.1.2's features, but had fundamental issues:

1. **Dummy Direction Labels**: Training dual outputs with `np.zeros_like(y_train)` - completely meaningless
2. **Unused Methods**: `prepare_data()` and `load_csv_data()` methods that were never called
3. **Missing Features**: Bull market detection that doesn't exist in V1 data
4. **Scaler Confusion**: Trying to use V1's feature scaler for target variable inverse transforms
5. **Mixed Logic**: V1.1.2's preprocessing pipeline doesn't apply to V1's pre-prepared numpy arrays

## Solution: Clean Rewrite

Rewrote V1.1 models to properly merge:
- **V1's workflow**: Load numpy arrays, work in scaled space, clean pipeline
- **v1.1.2's improvements**: Bidirectional architecture, dual outputs, custom biased loss

### Custom Biased Loss Function

```python
def custom_biased_mse(y_true, y_pred):
    """
    Penalizes under-predictions 2x more than over-predictions
    Encourages model to be slightly optimistic
    """
    error = y_true - y_pred
    under_prediction_weight = 2.0
    weights = tf.where(error > 0, under_prediction_weight, 1.0)
    return tf.reduce_mean(weights * tf.square(error))
```
